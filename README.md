# 🛡️ Разработка программного комплекса для интеллектуального анализа текстовых данных (RAG + Local LLM)

**Тема дипломного проекта:** Проектирование и разработка программного комплекса с веб-интерфейсом, обеспечивающего интеллектуальный анализ и извлечение информации из неструктурированных текстовых данных на основе архитектуры RAG (Retrieval-Augmented Generation) с использованием локально развернутых больших языковых моделей (LLM).

---

## 🎯 Концепция и цели проекта
Проект направлен на создание готового к эксплуатации инструмента, который автоматизирует работу с большими объемами документации, обеспечивая полную конфиденциальность и автономность обработки данных.

---

## 🏗 Архитектура AI-сервиса (Backend & AI)

*   **🔌 LlmServiceContract:** Сервисная архитектура и PHP-интерфейсы позволяют менять AI-провайдера через `.env` без правки бизнес-логики приложения.
*   **⏳ Асинхронный пайплайн (Laravel Jobs):** Генерация ответов вынесена в фоновые задачи (`ProcessAiResponse`), сохраняя интерфейс отзывчивым.
*   **🧠 Context-Aware RAG:** Трехуровневый поиск (База знаний + Short-term Memory + История чата) для корректного понимания уточняющих вопросов и разрешения анафор.
*   **📊 Векторный движок (pgvector):** Использование индекса **HNSW** с метрикой косинусного сходства для мгновенного поиска по чанкам (800 симв.).
*   **♻️ Жизненный цикл данных:** Поддержка **Soft Deletes** (скрытие векторов без перегенерации), версионность документов и транзакционный **Atomic Swap** при обновлении файлов.

---

## ⚡ Real-time взаимодействие и отказоустойчивость

*   **🛰 Мгновенная трансляция (Reverb + Echo):** Ответы ИИ из фоновых задач и системные уведомления передаются в интерфейс через WebSockets в реальном времени.
*   **📉 Асинхронный мониторинг (Inertia Lazy Props):** Статус доступности LLM-сервера подгружается независимо — основной интерфейс рендерится мгновенно, не блокируя пользователя.
*   **🔄 Реактивное обновление (Partial Reload):** При изменении состояний фронтенд точечно обновляет данные без перезагрузки страницы и потери фокуса.
*   **🏎 Оптимизация обмена данными:** Использование механизмов **Debounce** и обработки очередей на стороне клиента снижает нагрузку на сервер.

---

## 🎨 Интерфейс и пользовательский опыт (UX/UI)

Проект реализован как современное SPA (Single Page Application) с фокусом на эргономику:

*   **💅 Графическая оболочка:** Верстка на базе **Tailwind CSS 4** с компонентами **Headless UI** и анимациями на **Framer Motion**.
*   **📜 Интеллектуальный скролл:** Автоматическая прокрутка к новым сообщениям и сохранение позиции при чтении истории во время генерации ответа.
*   **🔔 Реактивный индикатор:** Плавающая кнопка быстрого спуска с уведомлением о новых данных, если пользователь находится в середине диалога.
*   **🗂 Контекстное управление:** Динамический список чатов с автоматическим именованием и мгновенным переключением сессий.
*   **📱 Адаптивность:** Полная оптимизация интерфейса для мобильных устройств и планшетов.
*   **⚠️ Безопасность взаимодействия:** Использование модальных окон для подтверждения критических операций (удаление чата, очистка истории).

## 🔐 Безопасность и управление данными

*   **🛡 Управление доступом:** Авто-назначение прав Admin через **Eloquent Observers**, защита админ-панели через **кастомный Middleware** и изоляция чатов через **Policies**.
*   **🔄 Cascade Smart Restoration:** Алгоритм каскадного восстановления связей (User → Chats → Messages) по временным меткам.
*   **♻️ Гибридное удаление:** Soft Delete для архивации пользователей и Force Delete для очистки системных ресурсов.
*   **🧪 Тестирование:** Использование **Mock-драйвера AI** для отладки UI-событий без нагрузки на LLM и расширенное логирование фоновых задач.

---

## 🧪 Тестирование и отладка

*   **🎭 Mock-драйвер AI-сервиса:** Имитация ответов нейросети и сбоев API для отладки WebSocket-событий и анимаций без нагрузки на LLM.
*   **🔍 Система диагностики:** Расширенное логирование (**Laravel Logging**) для мониторинга жизненного цикла тяжелых фоновых задач и взаимодействия с Ollama API.

## 🗺 Перспективы развития (Roadmap)

*   **🚀 Масштабирование поиска:** Миграция с pgvector на специализированную векторную СУБД (**Qdrant**).
*   **📈 Мониторинг инфраструктуры:** Внедрение **Laravel Horizon** для визуального контроля очередей и нагрузки.
*   **📊 Мультимодальный анализ:** Реализация **Text-to-SQL** запросов через Prism для прямой аналитики из БД.
*   **📱 Стриминг и UX:** Переход на потоковую генерацию ответов (Streaming) в реальном времени через WebSockets для отображения текста по мере его генерации моделью. Глубокая тач-оптимизация интерфейса для комфортной работы на мобильных устройствах.
*   **📝 Эволюция контента:** Поддержка **Markdown и LaTeX** для формул, таблиц и подсветки кода.

## 🚀 Развертывание

### Предварительные требования
*   PHP 8.4+, PostgreSQL 16 + pgvector.
*   Ollama (модели `mistral` и `nomic-embed-text`).

### Установка
1.  **Клонирование и зависимости:**
    ```bash
    git clone https://github.com
    cd project
    composer install && npm install
    ```
2.  **Настройка окружения:**
    ```bash
    cp .env.example .env
    php artisan key:generate
    ```
3.  **Миграции и импорт:**
    ```bash
    php artisan migrate

    📂 Подготовка данных: Поместите документы (.pdf, .docx, .txt) в директорию storage/app/private/init_kb/. Система автоматически распределит их по категориям на основе имен подпапок.

    php artisan kb:import  # Массовый импорт базы знаний
    ```
4.  **Запуск:**
    ```bash
    php artisan reverb:start &
    php artisan queue:work &
    npm run dev
    ```
